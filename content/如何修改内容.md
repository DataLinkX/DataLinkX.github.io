# 如何修改网站内容（给非技术用户）

网站的**所有文字内容**都集中在这个 `content/` 文件夹里，每个页面对应一个 `.md` 文件。
你**只需要改这些 `.md` 文件**，不用碰任何 HTML。改完保存、推送（push），网页会自动重新渲染。

## 哪个文件对应哪个页面

| 文件 | 页面 |
|------|------|
| `home.md` | 首页正文（“Background / Research at a Glance / Recent Work”） |
| `research.md` | 研究方向 + 论文列表 |
| `teaching.md` | 教学 |
| `people.md` | 团队成员 |
| `news.md` | 访问学者、报告、会议 |
| `joinus.md` | 加入我们 |
| `contact.md` | 联系方式 |

> 首页最上方的姓名、职称、照片、按钮不在 md 里（很少改动），如需修改在 `index.html` 中。

## 基本写法（Markdown）

这些文件用 Markdown 格式，规则很少，几分钟就能学会：

```
## 大标题        （一条深蓝下划线的章节标题，比如 “Selected Publications”）
### 中标题        （比如年份 “2026”，或小节标题）
#### 小标题        （红色小标签，比如 “Graduates”“Computational Biology”）

普通段落直接写文字即可，空一行表示分段。

- 无序列表项
- 再一项

1. 有序列表项（论文列表用这个，序号会自动编号）
1. 再一项（前面都写 1. 就行，网页会自动排成 1、2、3…）

**加粗文字**       （用于突出作者本人 H. Jiang）
*斜体文字*         （用于期刊名、物种名等）
[显示文字](网址)    （超链接）
```

## 论文列表怎么写（research.md）

每条论文是有序列表的一项，一行一篇。约定：

- `**H. Jiang**` —— 作者本人（自动加粗）
- `<u>学生名</u>` —— 学生作者（自动加红色点下划线）
- `[*期刊名*](论文网址)` —— 期刊名（自动变成斜体、可点击）
- `[code](/resources/文件名)` —— 代码/附件等资源链接
- `(**Cover Paper**)`、`(a sister journal to *Cell*)` —— 补充说明直接用括号写

**示例：**

```
### 2026
1. <u>B. Shen</u>#, …, W. Chen*, **H. Jiang***. 论文标题写这里. [*Nature Communications*](https://论文链接). [code](/resources/xxx.zip)
```

想加新一年，就在最上面加一个 `### 2027` 再往下列论文即可。

## 添加下载文件（PDF、代码 zip）

把文件放进网站的 `resources/` 文件夹，然后在 md 里用 `[显示文字](/resources/文件名)` 引用。
图片放进 `img/` 文件夹，用 `![说明](/img/文件名)` 插入。

## 改完怎么让网站更新

1. 保存修改的 `.md` 文件
2. 在项目文件夹里执行：
   ```
   git add .
   git commit -m "更新内容"
   git push
   ```
3. 等十几秒，网站自动更新完成。

> 也可以直接在 GitHub 网页上打开对应的 `.md` 文件、点铅笔图标在线编辑并保存，同样会自动更新，全程不用命令行。
